内置库

time库

一些概念

  • 计时起点
    1970年1月1日0时0分0秒，可以用time.gmtime(0)获得

  • 浮点数时间
    一般从计时起点开始计算

  • struct_time时间
    便于程序员使用的内部结构

    time.struct_time(tm_year=2020, tm_mon=1, tm_mday=25, tm_hour=3, tm_min=10, tm_sec=10, tm_wday=5)

    序号 属性名 取值范围
    0 tm_year 四位整数
    1 tm_mon [1,12]
    2 tm_mday [1,31]
    3 tm_hour [0,23]
    4 tm_min [0,59]
    5 tm_sec [0,61] 60仅用于调时
    6 tm_wday [0,6],星期一为0
    7 tm_yday [1,366]
    8 tm_isdst 0:DST,1:非DST,-1:随系统
    N/A tm_zone 时区的缩写表示
    N/A tm_gmtoff 与UTC时间相差的秒数
  • 字符串时间
    便于人看

时间获取库函数

  • time()
    返回一个从计时起点开始的表示时间的浮点数，返回一个时间戳
  • gmtime()
    返回一个struct_time表示的时间，如果是gmtime(s),s变成struct_time时间
  • ctime()
    返回一个字符串表示的时间，如果提供参数s，则把s变成字符串时间
  • localtime()
    返回一个struct_time表示的本地时间。localtime(s) 把s变成本地时间
  • mktime()
    将一个struct_time的本地时间t变成一个浮点数时间
  • asctime()
    返回一个字符串表示的时间，如果提供参数t，则把t变成字符串时间

时间格式化

  • strftime()
    f 把struct_time时间转换成字符串时间
    strftime(tpl,t) tpl是格式化模板字符串，定义输出效果，t是struct_time变量

    格式化 说明 范围和实例
    %Y 年份 0000-9999
    %m 月份 01-12
    %d 日期 01-31
    %A 星期 Monday-Sunday
    %a 星期缩写 Mon-Sun
    %H 小时 24h制 00-23
    %I 小时 12h制 01-12
    %p 上/下午 AM,PM
    %M 分钟 00-59
    %S 秒 00-59
    %Z 时区 字符串
    %% %字符 %

  • strptime()
    把字符串时间转换成struct_time时间
    strptime(ts,tpl)

re库

  • 正则表达式语法

    操作符 说明
    . 表示任何单个字符
    [] 字符集，对单个字符给出取值范围
    [^] 非字符集，对单个字符给出排除范围
    * 前一个字符0次或无限次出现
    + 前一个字符1次或无限次出现
    前一个字符0次或1次出现
    | 左右表达式任意一个
    {m} 扩展前一个字符m次
    {m,n} 扩展前一个字符m至n次
    ^ 匹配字符串开头
    $ 匹配字符串结尾
    () 分组标记，内部只能使用|操作符
    \d 数字
    \w 单词字符

  • search(pattern, string, flags=0)

    在一个字符串中搜索匹配正则表达式的第一个位置，返回match对象

    匹配的字符串可以通过match.goup(0)来查看

    pattern 正则表达式的字符串胡原生字符串表示

    string 待匹配字符串

    flags 正则表达式使用时的控制标记

    常用标记  re.I 忽略正则表达式的大小写 re.M 正则表达式中的^操作符能够将给定字符串的每行当作匹配的开始 re.S 正则表达式中的.操作符能够匹配所有字符

  • match(pattern, string, flags=0)
    在一个字符串的开始位置起的匹配正则表达式，返回match对象

  • findall(pattern, string, flags=0)

    搜索字符串，以列表类型返回全部能匹配的子串
    pattern 正则表达式的字符串或原生字符串表示

    string 待匹配字符串

    flags 正则表达式使用时的控制标记

  • split(pattern, stringm maxsplit=0, flags=0)

    将一个字符串按照正则表达式匹配结果进行分割，返回列表类型

    pattern 正则表达式的字符串或原生字符串表示

    string 待匹配字符串

    maxsplit 最大分割数，剩余部分作为最后一个输出

    flags 正则表达式使用时的控制标记

  • finditer(pattern, string, flags=0)

    搜索字符串，返回一个匹配结果的迭代类型，每个迭代元素是match对象

  • sub(pattern, repl, string, count=0, flags=0)

    在一个字符串中替换所有匹配正则表达式的子串，返回替换后的字符串

    repl 替换匹配字符串的字符串

    string 待匹配的字符串

    count 匹配的最大替换次数

    flags 正则表达式使用时的控制标记

  • compile(pattern)
    将一个正则串编译成一个对象，可调用上述的所有方法

  • match对象

    • .string 待匹配的文本
    • .re 匹配时使用的patter对象
    • .pos 正则表达式搜索文本的开始位置
    • .endpos 正则表达式搜索文本的结束位置
    • .group(0) 获取匹配后的字符串
    • .start() 匹配字符串在原始字符串的开始位置
    • .end() 匹配字符串在原始字符串的结束位置
    • .span() 返回(.span(), .end())

jieba库

中文分词

提供了三种分词模式，

精确模式:将文本精确地切开，分词后无冗余，适合文本分词
全模式: 把文本中所有可能成词的词语都扫描出来，速度快，不能解决歧义
搜索引擎模式: 在精确模式基础上，对长词再次切分，提高召回率。

  • jieba.cut(s) 精确模式，分词后返回一个迭代器，用for..in形式遍历结果
  • jieba.lcuts(s) 精确模式，分词后返回一个列表
  • jieba.cut(s, cut_all=True) 全模式，分词后返回一个迭代器
  • jieba.lcut(s, cut_all=True) 全模式
  • jieba.cut_for_search(s) 搜索引擎模式，分词后返回一个迭代器，用for..in 形式遍历结果
  • jieba.lcut_for_search(s) 搜索引擎模式，分词后返回一个列表
  • jieba.add_word(w) 向分词词典中增加新词w

WordCloud参数

  • width 指定词云对象生成图片的宽度，默认400像素
  • height 指定词云对象生成图片的高度，默认200像素
  • min_font_size 指定词云中字体的最小字号，默认4号
  • max_font_size 指定词云中字体的最大字号，根据高度自动调节
  • font_step 指定词云中字体的步进间隔，默认为1
  • font_path 指定字体文件的路径
  • max_words 指定词云显示的最大单词数量，默认为200
  • stop_words 指定词云的排除词列表
  • mask 指定词云形状，需要引用imread()函数，默认为长方形
  • backgroud_color 指定词云图片的背景颜色，默认为黑色

面向对象

三个特征

  • 封装
    属性和方法抽象，用数据和操作数据的方法来形成对象逻辑
    目标是形成一个类对外可操作属性和方法的接口
  • 继承
    代码复用的高级抽象，用对象之间的继承关系来形成代码复用
  • 多态
    方法灵活性的抽象，让对象的操作更加灵活，更多复用代码

类和对象

类 逻辑抽象和产生对象的模板，一组变量和函数的特定编排

对象 具体表达数据及操作的实体，相当于程序中的”变量”

实例化 从类到对象的过程。

继承

类继承的构建

class <类名>(<基类名>):
    def __init__(self, <参数列表>):
        <语句块>

基类的属性基本等同于定义在派生类中

派生类可以直接使用基类的类属性 实例属性

派生类可以直接使用基类的各种方法

使用基类的类方法和类属性时,要用基类的类名调用

class DemoClass:
    count = 0 # 定义类属性
    def __init__(self, name):
        self.name = name # 定义实例属性
        DemoClass.count += 1 # 对类属性进行操作
    def getName(self): # 定义实例方法
        return self.name
class HumanNameClass(DemoClass): # 继承DemoClass
    def printName(self):
        # 通过<基类名>.<属性名>调用类属性
        # 通过self.<属性名>
        return str(DemoClass.count) + self.name
dc1 = HumanNameClass("老王")
print(dc1.getName())
print(dc1.printName())

与继承判断有关的Python内置函数

  • issubclass(cls1, cls2) 判断类cls1是否是类cls2的子类, 返回True/False

最基础类

object类是Python所有类的基类,所有类定义时默认继承object类,保留属性和保留方法本质上是object类的属性和方法

对象三要素

  • 标识identity 对象一旦构建不会改变,用id()获得,一般是内存地址
  • 类型type 对象的类型,用type()获得
  • 值value 分为可变mutable与不可变immutable两种

内置功能

  • id(x) 返回x的标识
  • x is y 判断x和y的标识是否相等,返回True/False,不判断值

重载

派生类对基类属性或方法的再定义

  • 属性重载
    派生类定义并使用了与基类相同名称的属性
    采用最近覆盖原则

    • 优先使用派生类重定义的属性和方法
    • 然后寻找基类的属性和方法
    • 再寻找超类的属性和方法

  • 方法重载
    派生类定义并使用了与基类相同名称的方法

    • 完全重载
      派生类完全重定义与基类相同名称的方法，直接在派生类中定义同名方法即可。

    • 增量重载
      派生类扩展定义与基类相同名称的方法 

      class <派生类名>(<基类名>):
    def <方法名>(self, <参数列表>):
        super().<基类方法名>(<参数列表>)

类的引用

引用(reference) 对象的指针

引用是内存中真实对象的指针，表示为变量名或内存地址，每个对象存在至少一个引用，id()函数用于获得引用，在传递参数和赋值时，Python传递对象的引用，而不是复制对象

对于不可变对象，解释器为相同值维护尽量少的内存区域；对于可变对象，解释器为每个对象维护不同内存区域。

浅拷贝

仅复制最顶层对象的拷贝方式，默认拷贝方式

ls = ["Python", 123]
la = ls.copy()
lb = ls[:]
lc = list(ls)
for i in [ls, la, lb, lc]:
    for c in i:
        print(c, id(c), " ", end=" ")
    print(" ", i, id(i))

# 输出结果
# Python 913413848 123 915415464 ['Python', 123] 91113434
# Python 913413848 123 915415464 ['Python', 123] 91222222
# 列表被拷贝了，但是元素没有被拷贝

库、模块、包

库(Library)

一种对特定功能集合的通俗说法
包含一些程序功能，通过import引入使用，对应模块和包

模块

以单个文件为命令空间的代码片段
模块名就是文件名
本质是一个独立的、由模块名组织的命名空间
在import时一次性执行，但是对于单下划线的顶层命名元素，不会再from…import * 时被导入

  • 名称属性
    表达模块名称的预定义变量
    通过__name__ 获取
    当程序以脚本方式直接执行时，__name__的值为__main__
    当程序以模块方式被引用执行时，__name__的值为模块名称
    可以用来区分程序以何种方式执行

    # m.py文件
def _module_func():
    pass
print(__name__) 
# a.py文件
import m

# 当直接执行m.py文件时，输出结果为 __main__
# 当执行a.py文件时，输出结果为m
if __name__ == '__main__':
  # 作为模块主体功能的单元测试部分
    # 作为模块内部保留的额外功能部分

包(Package)

由一组模块构成，有层次命名空间的程序功能
每个包中一定有一个__init__.py文件

模块和包的构建

模块构建的原则

  • 功能闭包
    单一.py文件实现单一且完整的功能
  • 抽象适度
    用函数或类进行抽象，结合功能选择合适抽象
  • 操作闭包
    模块无顶层可执行语句，导入时无输出，尽量无全局变量

包的构建

  • 常规包
    通过__init__.py对文件和目录组织形成的包
    在包被导入的时候，__init__.py会被执行
  • 命名空间包
    由更分散子包组成的包
  • __all__属性
    from…import * 形式需要在__init__.py文件中增加__all__属性赋值
    __all__需要被赋值为列表对象，包含当前包下所有希望被导入的模块名称
    __all__用来辅助导入模块，但不能辅助导入列表
  • 常规包的构建流程
    设计好包层次结构(命名空间)，每个目录配置一个__init__.py文件
    完善__init__.py文件，并酌情赋值_all__属性
    再次理解包和模块的不同，模块提供功能，包提供命名空间
  • 常规包的导入流程
    每个包仅被导入一次，且包导入按照层次结构进行
    直接导入包不能调用功能，需要导入到模块层次
    from...import 直接导入具体模块，可以简化调用时命令空间表达
  • 命名空间包
    表达命名空间层次结构的一种逻辑包形式
    命名空间中各部分可以在不同的文件系统位置
    命名空间中各子包并不包含__init__.py文件(普通目录)
    Python解释器通过sys.path变量来隐式维护命令空间

    sys.path 指定搜索路径的字符串列表，路径是相对或绝对路径，可以用sys.path.append(p)增加新路径p，载入后，根据其中包的名称和层次结构自然组成了命名空间包
    __path__属性记录了某个包的绝对或相对路径
