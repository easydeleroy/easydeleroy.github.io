yuuwsgi配置与安装

无标签
发布日期:   2020-05-27
文章字数:   704

基本概念

uWSGI是一个具有超快的性能、超低的内存占用、详细的日志以及多应用管理等特点的使用纯C语言编写的服务器。它实现了WSGI协议、uwgi协议、http协议等。

安装

使用pip3 install uwsgi命令来进行安装，当出现Successfully install uwsgi-2.0.18时表示安装成功。如下图。

测试

为了测试安装的uwsgi是否可以正常使用，我们可以编写一个简单的应用程序来对它进行测试。

首先我们编写一个简单的应用程序代码。

# server03.py
def application(environ, start_response):
  start_response('200 OK',[('Content-Type','text/html')])
  return ['<h1>Hello World'.encode('utf-8')]

然后运行 uwsgi --http :13579 --wsgi-file server03.py,其中--http参数指定了监听的端口号,--wsgi-file指定了应用程序名。运行后回返回在终端上输出一些环境信息，包括服务器的名字、CPU核数、当前工作路径等。结果如下图

在浏览器上打开对应的网页便可显示编写的应用程序中的响应结果。

使用配置文件启动

uWSGI除了可以使用命令行的方式来配置外，还可以通过一个配置文件来配置。

uWSGI有很多的配置参数，常用的主要有

  • http：指定端口号
  • wsgi-file：指定服务器端的程序名
  • master：启用主进程
  • socket：以WSGI的Socket方式运行，需要指定连接地址和端口，在使用代理服务器时使用
  • chdir：指定uWSGI的目录
  • processes: 指定启动服务器的进程数
  • threads：指定每个服务器的线程数
  • uid：指定运行uWSGI的Linux用户id
  • max-requests：指定最大的请求数
  • daemonize：指定日志文件路径
  • log-maxsize：指定日志文件的最大存储
  • pidfile：当以后台形式运行时，uwsgi服务的id，用来终止服务
  • module：配置Django时用来指定module文件的路径

比如，如下配置文件(uwsgi.ini)，指定了端口号、服务器端的程序名、进程和线程数、日志文件路径和最大存储、服务ID存储路径。

[uwsgi]
chdir =/home/ubuntu/
wsgi-file = server03.py
http-socket = :8000
processes = 4
threads = 1
daemonize = %(chdir)/log/uwsgi-8000.log
log-maxsize = 1024*1024*1024
pidfile = %(chdir)/pid/uwsgi-8000.pid

在终端上输入uwsgi uwsgi.py 文件来启动服务,结果如下图。

其他说明

当uwsgi服务已后台形式运行时，可以用过uwsgi --stop ID名来进行终止服务。

当需要判断某个端口是否被占用时，可以通过lsof -i:端口号来查看
