准备

这一节，我们要将前几章中介绍的项目部署到服务器上，并能访问。

首先我们需要在租一台云服务器，我这里购买的阿里云的服务器。打开https://www.aliyun.com/，点击产品分类，选择云服务器ECS，点击立即购买。

这里我们选择任何一个地区的服务器都可以，选择配置以及系统，进行购买即可，如下图。

购买完成后，我们需要支持SSH连接的软件连接服务器。我这里使用的是xshell，如下图。

在主机位置输入在阿里云购买的服务器的公网IP，如下图，并按照提示输入用户名和密码，(注意，阿里云服务器的用户名一般为ubuntu或root)。

连接成功后，如下图。

环境搭建

服务器已经购买完成了，并且成功的进行了远程连接。接下来我们就需要搭建Django的运行环境。

我们选择的Django的运行环境是以Mysql作为数据库、Nginx作为代理服务器、uWSGI作为服务器，另外我们的项目存放在/home/下。

安装并配置Mysql

首先先更新下我们刚刚购买的服务器里的软件，输入sudo apt-get updateh和sudo apt-get upgrade，完成后，就可以安装Mysql了，输入sudo apt-get install mysql-client mysql-server libmysqlclient-dev libmysqld-dev安装Mysql的客户端和服务端以及依赖。等待安装完成，通过service mysql status来查看Mysql的状态，如果出现如下内容，则说明Mysql安装并启动成功，如果出现如第二张图的内容，则说明Mysql没有启动，可以通过service mysql start来启动。通过mysql -uroot -p命令来登录Mysql。

进入后，我们首先要修改Mysql的root用户的密码，当然，也可以创建一个新的用户。输入use mysql,来选择mysql库，输入update mysql.user set authentication_string=password('root') where user='root' and Host ='localhost';和update user set plugin="mysql_native_password";，将root用户的密码修改为root，最后通过sudo service mysql restart命令，重启Mysql服务，如下图。

因为在使用Django访问数据库时，我们使用的是以ORM形式来访问的，它可以根据映射关系将数据库中数据表创建好，但是不能创建库，所以我们需要自己去创建库。进入Mysql，输入create database blog charset=utf8创建一个编码为utf-8格式的数据库。

上传项目

我们要将我们开发好的Django项目上传到服务器上，需要使用支持FTP文件传输的软件将我们的项目传输到服务器上。当然，也可以直接在服务器上使用git clone https://gitee.com/easyleroy/django-blog.git 来直接下载我们上传到码云上的项目进行部署，如下图。

Django运行

收集静态文件

在项目的根目录下输入python3 manager collectstatic来收集静态文件。

创建迁移文件并映射到数据库中

输入python3 manage.py makemigrations创建迁移文件。
