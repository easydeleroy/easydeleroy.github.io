模板语法

前端渲染

简单来说，将数据填充到HTML标签中，就是前端渲染。将模板与数据结合，渲染成静态的HTML内容。比如art-template模板引擎、Vue等。

指令

指定的本质就是自定义属性，用在标签中的，比如data-name等。
指令的格式:以v-开始(比如:v-cloak),指令是vue框架设计者已经规定好的。
官网-指令

v-text
  • 作用
    将数据填充到标签中,与插值表达式一样，但不存在闪动问题
  • 使用
    <div v-text='msg'></div> msg为data中的变量
v-html
  • 作用
    将HTML内容填充到标签中
  • 使用
    <div v-text='msg1'></div> msg1为data中的变量
v-pre

  • 作用
    用于显示原始信息，会跳过编译过程

  • 使用

    <div v-pre></div> 网页上显示

v-cloak
  • 作用
    解决插值表达式存在的闪动问题
  • 原理
    先将元素隐藏，再替换为正确值后再显示
  • 使用
    1. 提供样式[v-cloak]{display:none;}
    2. 在插值表达式所在的标签中添加v-cloak指令
v-once

  • 数据响应式
    数据发生变化，导致页面内容变化

  • 作用
    对数据只编译一次，后面对数据的修改，不会改变网页上的显示内容，即不存在数据响应式

v-model

  • 双向数据绑定
    页面与数据可相互影响，进行改变

  • 使用

    

    
        
    
            
    {{msg}}

事件绑定

v-on

  • 作用
    进行事件绑定

  • 使用

    <input v-on:click="" />

  • 简写
    <input @click="" />

  • 参数传递方式一
    <button @click="say("hi",$event)"> 这里的$event代表的是事件对象，放到最后一个参数

  • 参数传递方式二
    <button @click="say"> 这时，自动会将事件对象传递过去

  • 事件修饰符

    • .stop 阻止冒泡
      <a v-on:click.stop="handle"></a>
    • .prevent 阻止默认行为
      <a v-on:click.prevent="handle"></a>
    • 更多
      修饰符

  • 按键修饰符

    • .enter 回车
      <input v-on:keyup.enter="submit">

    • .delete删除键
      <input v-on:keyup.delete="submit">

    • 自定义按键修饰符
      设置全局Vue.config.keyCodes.f1 = 112

      或直接使用<input v-on:keyup.[按键对应的数字]="submit">

      按键对应的数字可通过event.keyCode查看

    • 更多
      按键修饰符

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Document</title>
    <style type="text/css">
        .tab ul {
            overflow: hidden;
            padding: 0;
            margin: 0;
        }
        .tab ul li {
            box-sizing: border-box;
            padding: 0;
            float: left;
            width: 100px;
            height: 45px;
            line-height: 45px;
            list-style: none;
            text-align: center;
            border-top: 1px solid blue;
            border-right: 1px solid blue;
            cursor: pointer;
        }
        .tab ul li:first-child {
            border-left: 1px solid blue;
        }
        .tab ul li.active {
            background-color: orange;
        }
        .tab div {
            width: 500px;
            height: 300px;
            display: none;
            text-align: center;
            font-size: 30px;
            line-height: 300px;
            border: 1px solid blue;
            border-top: 0px;
        }
        .tab div.current {
            display: block;
        }
    </style>
</head>
<body>
    <div id="app">
        <p>
            增加tab栏 <input type="text">
        </p>
        <div class="tab">
            <ul>
                <li v-on:click='change(index)' :class='currentIndex==index?"active":""' :key='item.id' v-for='(item,index) in list'>{{item.title}}</li>
            </ul>
            <div :class='currentIndex==index?"current":""' :key='item.id' v-for='(item, index) in list'>
                <img :src="item.path">
            </div>
        </div>
    </div>
    <script type="text/javascript" src="js/vue.js"></script>
    <script type="text/javascript">
        var vm = new Vue({
            el: '#app',
            data: {
                currentIndex: 0, // 选项卡当前的索引
                list: [{
                    id: 1,
                    title: 'apple',
                    path: 'img/apple.png'
                }, {
                    id: 2,
                    title: 'orange',
                    path: 'img/orange.png'
                }, {
                    id: 3,
                    title: 'lemon',
                    path: 'img/lemon.png'
                }]
            },
            methods: {
                change: function(index) {
                    // 在这里实现选项卡切换操作：本质就是操作类名
                    // 如何操作类名？就是通过currentIndex
                    this.currentIndex = index;
                }
            }
        });
    </script>
</body>
</html>

计算属性

  • 作用
    表达式的计算逻辑可能会比较复杂，使用计算属性可以使模板内容更加简洁

  • 用例

    


  
    
    
    {{msg}}
    
    
    {{reverseString}}

  • 计算属性和方法的区别
    计算属性是基于他们的依赖进行缓存的，方法不存在缓存，将计算结果会放在缓存中，这样就减少了计算。

修改响应式数据

使用索引的方式修改数组的数据不是响应式的，应该使用Vue.set方法

Vue.set(vm.item,indexOfItem,newValue)
参数一: 表示要处理的数组名称
参数二:表示要处理的数组索引
参数三:表示要处理的数组的值
