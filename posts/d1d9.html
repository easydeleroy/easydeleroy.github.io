MVVM设计思想

基本使用

从”Hello World”开始

  1. 需要提供标签用于填充数据
  2. 引入vue.js库文件
  3. 可以使用vue的语法做功能
  4. 把vue提供的数据填充到标签里面
<!DOCTYPE html>
<html lang="en">
<body>
    <div id="app">
        <div>{{msg}}</div>  <!--差值表达式:将数据填充到HTML标签中，并可以实现一些简单的计算操作-->
        <div>{{1 + 2}}</div>
        <div>{{msg + '----' + 123}}</div>
    </div>
    <script type="text/javascript" src="js/vue.js"></script>
    <script type="text/javascript">
        var vm = new Vue({
            el: '#app', // 元素的挂载位置
            data: { // 模型数据
                msg: 'Hello World!'
            }
        });
    </script>
</body>
</html>

模板语法

前端渲染

简单来说，将数据填充到HTML标签中，就是前端渲染。将模板与数据结合，渲染成静态的HTML内容。比如art-template模板引擎、Vue等。

指令

指定的本质就是自定义属性，用在标签中的，比如data-name等。
指令的格式:以v-开始(比如:v-cloak),指令是vue框架设计者已经规定好的。
官网-指令

v-text
  • 作用
    将数据填充到标签中,与插值表达式一样，但不存在闪动问题
  • 使用
    <div v-text='msg'></div> msg为data中的变量
v-html
  • 作用
    将HTML内容填充到标签中
  • 使用
    <div v-text='msg1'></div> msg1为data中的变量
v-pre

  • 作用
    用于显示原始信息，会跳过编译过程

  • 使用

    <div v-pre></div> 网页上显示

v-cloak

  • 作用
    解决插值表达式存在的闪动问题

  • 原理
    先将元素隐藏，再替换为正确值后再显示

  • 使用

    1. 提供样式[v-cloak]{display:none;}
    2. 在插值表达式所在的标签中添加v-cloak指令

  • 代码

    <!DOCTYPE html>
<html lang="en">
<head>
  <style type="text/css">
  [v-cloak]{
    display: none;
  }
  </style>
</head>
<body>
  <div id="app">
    <div v-cloak>{{msg}}</div>
  </div>
  <script type="text/javascript" src="js/vue.js"></script>
  <script type="text/javascript">
    var vm = new Vue({
      el: '#app',
      data: {
        msg: 'Hello Vue'
      }
    });
  </script>
</body>
</html>
v-once

  • 数据响应式
    数据发生变化，导致页面内容变化

  • 作用
    对数据只编译一次，后面对数据的修改，不会改变网页上的显示内容，即不存在数据响应式

v-model

  • 双向数据绑定
    页面与数据可相互影响，进行改变

  • 使用

    <!DOCTYPE html>
<html lang="en">
    <body>
        <div id="app">
            <div>{{msg}}</div>
            <div>
                <input type="text" v-model='msg'>
            </div>
        </div>
        <script type="text/javascript" src="js/vue.js"></script>
        <script type="text/javascript">
            var vm = new Vue({
                el: '#app',
                data: {
                    msg: 'Hello Vue'
                }
            });
        </script>
    </body>
</html>

事件绑定

v-on

  • 作用
    进行事件绑定

  • 使用

    <input v-on:click="" />

  • 简写
    <input @click="" />

  • 参数传递方式一
    <button @click="say("hi",$event)"> 这里的$event代表的是事件对象，放到最后一个参数

  • 参数传递方式二
    <button @click="say"> 这时，自动会将事件对象传递过去

  • 事件修饰符

    • .stop 阻止冒泡
      <a v-on:click.stop="handle"></a>
    • .prevent 阻止默认行为
      <a v-on:click.prevent="handle"></a>
    • 更多
      修饰符

  • 按键修饰符

    • .enter 回车
      <input v-on:keyup.enter="submit">

    • .delete删除键
      <input v-on:keyup.delete="submit">

    • 自定义按键修饰符
      设置全局Vue.config.keyCodes.f1 = 112

      或直接使用<input v-on:keyup.[按键对应的数字]="submit">

      按键对应的数字可通过event.keyCode查看

    • 更多
      按键修饰符

  • 用例

    <!DOCTYPE html>
<html lang="en">
<body>
    <div id="app">
        <div>{{num}}</div>
        <div>
            <button v-on:click='num++'>点击</button>
            <button @click='num++'>点击1</button>
            <button @click='handle'>点击2</button>
            <button @click='handle()'>点击3</button>
        </div>
    </div>
    <script type="text/javascript" src="js/vue.js"></script>
    <script type="text/javascript">
        var vm = new Vue({
            el: '#app',
            data: {
                num: 0
            }, // 注意这里不要忘记加逗号 
            // methods中主要是定义一些函数
            methods: {
                handle: function() {
                    // 这里的this是Vue的实例对象
                    console.log(this === vm)
                    this.num++;//在函数中 想要使用data里面的数据一定要加this
                }
            }
        });
    </script>
</body>
</html>

class样式处理

  • 对象语法
    <div v-bind:class="{active:isActive}"></div>

  • 数组语法
    <div v-bind:class="[activeClass,errorClass]"></div>

  • 用例

    <!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Document</title>
  <style type="text/css">
    .active {
      border: 1px solid red;
      width: 100px;
      height: 100px;
    }
    .error {
      background-color: orange;
    }
  </style>
</head>
<body>
  <div id="app">
    <div v-bind:class="{active: isActive,error: isError}">
      测试样式1
    </div>
    <button v-on:click='handle1'>切换</button>
     <div v-bind:class='[activeClass, errorClass]'>测试样式2</div>
      <button v-on:click='handle2'>切换</button>
  </div>
  <script type="text/javascript" src="js/vue.js"></script>
  <script type="text/javascript">
    var vm = new Vue({
      el: '#app',
      data: {
        isActive: true,
        isError: true,
        activeClass: 'active',
        errorClass: 'error'
      },
      methods: {
        handle1: function(){
            this.isActive = !this.isActive;
            this.isError = !this.isError;
        },
        handle2: function(){
            this.activeClass = '';
            this.errorClass = '';
        }
      }
    });
  </script>
</body>
</html>

自定义指令

  • 语法规则

    Vue.directive('指令名',{
    inserted: function(el) { // 钩子函数
        el.focus(); // el表示指令所绑定的元素
    }
})

  • 自定义指令的钩子函数

    • bind ： 只调用一次，指令第一次绑定到元素 时调用，用这个钩子函数可以定义一个在绑定时执行一次的初始化动作。
    • inserted ： 被绑定元素插入父节点时调用（父节点存在即可调用，不必存在于 document 中） ．
    • update ： 被绑定元素所在的模板更新时调用，而不论绑定值是否变化。通过比较更新前后的绑定值，可以忽略不必要的模板更新。
    • componentUpdated ： 被绑定元素所在模板完成一次更新周期时调用．
    • unbind ： 只调用一次，指令与元素解绑时调 用 。

  • 带参数的自定义指令

    Vue.directive('color', {
    inserted: function(el, binding) {
        el.style.backgroundColor = binding.value.color;
    }
})

//使用
<input type="text" v-color='{color:"orange "}'>

  • 钩子函数的参数

    • el 指令所绑定的元素，可以直接操作DOM
    • binding 一个对象，包含以下属性
      • name 指令名。不包含前缀v-
      • value 指令的绑定值，比如v-directive=”1+1” value的值为2
      • oldValue 指令绑定的前一个值，仅在update和componentUpdated钩子中可用
      • expression 绑定值的字符串形式
      • arg 传给指令的参数
      • modifiers 一个包含修饰符的对象
    • vnode Vue编辑器生成的虚拟节点
    • oldVnode 上一个虚拟节点仅在update和componentUpdated钩子可用

  • 用例

    <!DOCTYPE html>
<html lang="en">
<body>
  <div id="app">
    <input type="text" v-color='msg'>
  </div>
  <script type="text/javascript" src="js/vue.js"></script>
  <script type="text/javascript">
    Vue.directive('color', {
      bind: function(el, binding){
        el.style.backgroundColor = binding.value.color;
      }
    });
    var vm = new Vue({
      el: '#app',
      data: {
        msg: {
          color: 'blue'
        }
      }
    });
  </script>
</body>
</html>

计算属性

  • 作用
    表达式的计算逻辑可能会比较复杂，使用计算属性可以使模板内容 简洁

  • 用例

    <!DOCTYPE html>
<html lang="en">
<body>
  <div id="app">
    <div>{{msg}}</div>
    <div>{{reverseString}}</div>
  </div>
  <script type="text/javascript" src="js/vue.js"></script>
  <script type="text/javascript">
    var vm = new Vue({
      el: '#app',
      data: {
        msg: 'Nihao'
      },
      computed: {
        reverseString: function(){
          return this.msg.split('').reverse().join('');
        }
      }
    });
  </script>
</body>
</html>

  • 计算属性和方法的区别
    计算属性是基于他们的依赖进行缓存的，方法不存在缓存，将计算结果会放在缓存中，这样就减少了计算。

侦听器

  • 作用
    用于监听数据的变化，当数据发生变化时，就会通知侦听器所绑定的方法

  • 应用场景
    数据变化时，执行异步或开销较大的操作

  • 用例

    <!DOCTYPE html>
<html lang="en">
<body>
  <div id="app">
    <div>
      <span>用户名：</span>
      <span>
        <input type="text" v-model.lazy='uname'>
      </span>
      <span>{{tip}}</span>
    </div>
  </div>
  <script type="text/javascript" src="js/vue.js"></script>
  <script type="text/javascript">
    var vm = new Vue({
      el: '#app',
      data: {
        uname: '',
        tip: ''
      },
      methods: {
        checkName: function(uname) {
          // 调用接口，但是可以使用定时任务的方式模拟接口调用
          var that = this;
          setTimeout(function(){
            // 模拟接口调用
            if(uname == 'admin') {

变异方法

会影响数组的原始数据的变化

  • push() 向数组的末尾添加一个或多个元素
  • pop() 删除数组的最后一个元素并返回删除的元素
  • shift() 把数组的第一个元素从其中删除，并返回第一个元素的值
  • unshift() 向数组的开头添加一个或更多元素，并返回新的长度
  • splice() 添加或删除数组中的元素
  • sort() 对数组的元素进行排序
  • reverse() 反转数组的元素顺序

替换数组

不会影响原始的数组数据，而是形成一个新的数组

  • filter() 检测数值元素，并返回符合条件所有元素的数组
  • concat() 连接两个或更多的数组，并返回结果。
  • slice() 选取数组的的一部分，并返回一个新数组。

数组的一些操作

  • 遍历并修改一些信息

var book = this.books.filter(function(item){
