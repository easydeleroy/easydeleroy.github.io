十二、数组

概念

存储数据长度固定的容器，保证多个数据的数据类型要一致。

特点

  • 数组是一种引用数据类型
  • 数组当中的多个数据，类型必须统一
  • 数组的长度在程序运行期间不可改变

定义

  • 方式一：数据类型[] 数组名 = new 数据类型[长度]
  • 方式二：数据类型[] 数组名 = new 数据类型[](元素1,元素2....)
  • 方式三：数据类型[] 数组名 = {元素1,元素2,元素3....}
  • 注意：方式三不能拆分成两个步骤,方式一和方式二都可以。int[] a; a = new int[](1,2,3);
  • 注意：使用方式一创建的数组，会给里面的元素一个默认值：
    • 如果是整数类型，那么默认为0
    • 如果是浮点类型，那么默认为0.0
    • 如果是字符类型，那么默认为’\u0000’
    • 如果是布尔类型，那么默认为false
    • 如果是引用类型，那么默认为null

访问

  • 直接使用数组名[索引]访问数组元素
  • 数组的length属性来获取数组的长度

数组原理内存

内存

内存是计算机中的重要原件，临时存储区域，作用是运行程序。

JVM的内存划分

区域名称 作用
寄存器 给CPU使用，和我们开发无关
栈(stack) 存放的都是方法中的局部变量，一旦超出作用域，立刻从栈内存当中消失。方法的运行在栈当中
堆(heap) 凡是new出来的东西，都在堆当中。堆内存里面的东西都有一个地址值。
方法区(method area) 存储.class相关信息，包含方法的信息
本地方法栈(Native Method Stack) 与操作系统相关

数组在内存中存储

new出来的内存，都是在堆内存中存储的，方法中的变量arr保存的是数组的地址。

程序执行流程

  1. main方法进入方法栈执行
  2. 创建数组，JVM会在堆内存中开辟空间，存储数组
  3. 数组在内存中会有自己的内存地址，以十六进制数表示
  4. 数组中有3个元素，默认值0
  5. JVM将数组的内存地址赋值给引用类型变量arr
  6. 变量arr保存的是数组内存中的地址，而不是一个具体的数组，因此称为引用数据类型

数组的常见操作

获取数组最大值

public static void main(String[] args){
    int[] arr = {5, 15, 2000, 10000, 100, 3000};
    // 定义变量，保存数组中0的索引值
    int max = arr[0];
    // 遍历数组，取出每个元素
    for(int i = 0; i < arr.length; i++) {
        if(max < arr[i]){
            max = arr[i];
        }
    }
    System.out.println("数组中最大值是:" + max);
}

数组反转

public static void main(String[] args){
    int[] arr = {1, 2, 3, 4, 5};
    for(int min = 0, max = arr.length - 1; min <= max; min++, max--){
        int temp = arr[min];
        arr[min] = arr[max];
        arr[max] = temp;
    }
    // 反转后遍历数组
    for(int i = 0; i < arr.length; i++){
        System.out.println(arr[i]);
    }
}

数组作为方法参数

数组作为方法参数传递，传递的参数是数组内存的地址。

public static void main(String[] args){
    int[] arr = {1, 2, 3, 4};
    printArray(arr); // 传递的是地址
}
// 遍历数组打印输出
public static void printArray(int[] arr) { // 接收到也是地址
    for(int i = 0; i < arr.length; i++) {
        System.out.println(arr[i]);
    }
}

数组作为方法返回值

数作为方法返回值，返回的也是数组的内存地址

public static void main(String[] args){
    int[] arr = getArray();
    for(int i = 0; i < arr.length; i++){
        System.out.println(arr[i]);
    }
}

// 定义一个方法，返回数组
public static void getArray() {
    int[] arr = {1, 2, 3, 4};
    return arr; // 返回的是数组的内存地址
}

十三、面向对象

面向对象思想

java是一种面向对象的程序设计语言，对象泛指现实中一切事物，每个事物都具备自己的属性和行为。面向对象思想就是在计算机程序设计过程中，参照现实中事物，将事物的属性特征，行为特征抽象出来，描述成计算机事件的设计思想。

它有三大基本特征：继承、封装、多态。

类

一组相关属性（该事物的状态信息）和行为（该事物能做什么）的集合，可以看成是一类事物的模板，使用事物的属性特征和行为特征来描述该类事物。

创建类

public class ClassName {
    // 成员变量 对象事物的属性
    String name;
    int age;
    // 成员方法 对象事物的行为，和前面定义方法一样，只不过把static去掉
    public void study() {
        System.out.println("好好学习，天天向上！");
    }
}

成员变量和局部变量的区别

public class ClassName {
    String color; // 成员变量
    public void drive() {
        int speed = 80; // 局部变量
    }
}
  • 在类中的位置不同： 成员变量在类内，方法外；局部变量在方法中或者形参
  • 作用范围不同：成员变量在类内；局部变量在方法中。
  • 初始值的不同：成员变量会有默认值；局部变量没有，默认值，必须先定义，赋值，最后使用。
  • 在内存中的位置不同：成员变量在堆内存中，局部变量在栈内存中。
  • 生命周期不同，成员变量随着对象的创建而存在，随着对象的消失而消失；局部变量随着方法的调用而存在，随着方法的调用完成而消失。

标准的类

一个标准的类通常包含四个部分

  • 所有的成员变量都要使用private关键字修饰
  • 为每个成员变量编写一对Getter/Setter方法
  • 编写一个无参数的构造方法
  • 编写一个全参数的构造方法

对象

一类事物的具体提箱，对象是类的一个实例，必然具备该类事物的属性和行为。

创建对象

类名 对象名 = new 类名();

访问成员

对象名.成员变量;对象名.成员方法();

匿名对象

创建对象时，只有创建对象的语句，却没有把对象地址值赋值给某个变量，虽然是创建对象的简化写法，但是应用场景非常有限。

  • 应用在创建匿名对象后，直接调用方法。
  • 匿名对象可以作为方法的参数和返回值

类和对象的关系

类是对一类事物的描述，是抽象的。

对象是一类事物的实例，是具体的。

类是对象的模板，对象是类的实体。

封装性

原则

将属性隐藏起来，若需要访问某个属性，提供公共方法对其访问。

步骤

  1. 使用private关键字来修饰成员变量
  2. 对需要访问的成员变量，提供对应的一对getXxx和setXxx方法。（注意，对应boolean类型的，没有getXxx，而要用isXxx）

关键字 private

可以修饰成员变量和成员方法，修饰的成员变量和成员方法只能在本类内访问。

public class Student {
    private String name;
    private int age;

    public String getName() {
        return name;
    }
    public void setName(name) {
        this.name = name
    }
}

关键字 this

this代表所在类的当前对象的引用(地址)，记住，方法被哪个对象调用，方法中的this就代表那个对象。

构造方法

当一个对象被创建的时候，构造方法用来初始化该对象，给对象的成员变量赋初始值。（无论是否自定义构造方法，所有的类都有构造方法。）

public class Student {
    private String name;
    private int age;
    // 无参数的构造方法
    public Student() {}
    // 有参数的构造方法
    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    } 
}

static关键字

可以用来修饰的成员变量和成员方法，被修饰的成员是属于类的，而不是单单的属于某个对象的。

类变量

当static修饰成员变量时，该变量称为类变量，该类的每个对象都共享同一个类变量的值。任何对象都可以更改该类变量的值，但也可以在不创建该类的对象的情况下对类变量进行操作。

静态方法

当static修饰成员方法时，该方法称为类方法。静态方法在声明中有static，建议使用类名来调用，而不需要创建类的对象。

注意：

  • 静态方法可以直接访问类变量和静态方法。
  • 静态方法不能直接访问普通成员变量或成员方法，反之，成员方法可以直接访问类变量或静态方法。
  • 静态方法中不能使用this关键字
  • 静态方法只能访问静态成员

它存储在一块固定的内存区域（静态区），它优先于对象存在，被所有对象共享。

静态代码块

定义在成员位置，使用static修饰的代码块{}。它随着类的加载而执行且执行一次，优先于main方法和构造方法的执行。可以用来给类变量进行初始化赋值。

继承性

子类继承父类的属性和行为，使得子类对象具有与父类相同的属性，相同的行为，子类可以直接访问父类的非私有的属性和行为。它主要解决的问题是共性的抽取。子类除了有父类的内容，还有自己独有的内容。

继承的格式

class 父类 {
    ...
}
class 子类 extends 父类 {
    ...
}

特点

  • Java只支持单继承，不支持多继承
  • 顶层父类是Object类,所有的类默认继承Object
  • 子类和父类是一种相对的概念

继承后的特点–成员变量

子父类中出现了同名的成员变量时，在子类中需要访问父类中非私有成员变量时，需要使用super关键字，修饰父类成员变量，类似于之前学过的this.

继承后的特点–成员方法

如果子父类中出现重名的成员方法，这时的访问是一种特殊情况，叫做方法重写。如果要访问父类的方法，也是使用super

方法重写：子类中出现于父类一模一样的方式时，会出现覆盖效果。

继承后的特点-构造方法

  • 构造方法的名字是和类一样的，所以子类是无法继承父类的构造方法，但是在子类初始化的过程中，必须先执行父类的初始化动作，子类的构造方法中默认有一个super()，表示调用父类的构造方法。
  • 子类的每个构造方法中均有默认的super()，调用父类的空参构造。手动调用父类构造会覆盖默认的super()
  • super()和this() 都必须是在构造方法的第一行，所以不能同时出现
  • 在重写时，要保证父子类之间的方法的名称相同，参数列表也相同，@override写在方法前面，用来检测是不是有效的正确覆盖重写。
  • 子类方法的返回值必须小于等于父类方法的返回值范围。
  • 子类方法的权限必须大于等于父类方法的权限 (public > protected > default > private)

抽象性

父类中的方法，被它的子类们重写，子类各自的实现都不尽相同。那么父类的方法声明和方法主体，只有声明还有
意义，而方法主体则没有存在的意义了。我们把没有方法主体的方法称为抽象方法。Java语法规定，包含抽象方法
的类就是抽象类 。

定义

  • 抽象方法：没有方法体的方法
  • 抽象类：包含抽象方法的类

抽象方法

使用abstract关键字修饰方法，该方法就成了抽象方法，抽象方法只包含一个方法名，没有方法体。

修饰符 abstract 返回值类型 方法名 (参数列表);

抽象类

如果一个类包含抽象方法，那么该类必须是抽象类。抽象类也可以没有抽象方法。

abstract class 类名{}

抽象的使用

继承抽象类的子类必须重写父类所有的抽象方法（也叫实现方法），否则该子类也必须声明为抽象类。

注意事项

  • 抽象类不能创建对象
  • 抽象类中，可以有构造方法，是供子类创建对象时，初始化父类成员使用的。
  • 抽象类中，不一定包含抽象方法，但是有抽象方法的类必定是抽象类
  • 抽象类的子类，必须重写抽象父类中所有的抽象方法，否则，编译无法通过而报错。

多态性

多态是指同一行为，具有多个不同表现形式。

体现

父类引用指向子类对象

父类名称 对象名 = new 子类名称() 或 接口名称 对象名 = new 实现类名称()

当使用多态方式调用方式时，首先检查父类中是否有该方法，如果没有，则编译错误，如果有，执行的是子类重写后的方法。

多态的好处

实际开发过程中，父类类型作为方法形式参数，传递子类对象给方法，进行方法的调用，更能体现出多态的扩展性。

// 定义父类
public abstract class Animal {
    public abstract void eat();
}

// 定义子类
class Cat extends Animal {
    public void eat() {
        System.out.println("吃鱼");
    }
}
class Dog extends Animal {
    public void eat() {
        System.out.println("吃骨头");
    }
}

// 定义测试类
public class Test {
    public static void main(String[] args) {
        // 多态形式，创建对象
        Cat c = new Cat();
        Dog d = new Dog();

        // 调用showCatEat
        showCatEat(c);
        // 调用showDogEat
        showDogEat(d);
        /*
        以上两个方法, 均可以被showAnimalEat(Animal a)方法所替代
        而执行效果一致
        */
        showAnimalEat(c);
        showAnimalEat(d);
    }

    public static void showCatEat (Cat c){
        c.eat();
    }
    public static void showDogEat (Dog d){
        d.eat();
    }
    public static void showAnimalEat (Animal a){
        a.eat();
    }
}

由于多态特性的支持，showAnimalEat方法的Animal类型，是Cat和Dog父类类型，父类类型接收子类对象，当然可以把Cat对象和Dog对象，传递给方法。

当eat方法执行时，多态规定，执行的是子类重写的方法，那么效果自然与showCatEat、showDogEat方法一致，
所以showAnimalEat完全可以替代以上两方法。

不仅仅是替代，在扩展性方面，无论之后再多的子类出现，我们都不需要编写showXxxEat方法了，直接使用
showAnimalEat都可以完成。

所以，多态的好处，体现在，可以使程序编写的更简单，并有良好的扩展。

引用类型转换

  • 向上转型 多态本身是子类类型向父类类型向上转换的过程。
  • 向下转型 父类向子类转换，强制型的。

当使用多态方式调用方法时，首先检查父类中是否有该方法，如果没有，则编译错误，也就是说，不能调用子类拥有而父类没有的方法。所以想要调用子类特有的方法，必须做向下转型。

instanceof关键字

变量名 instanceof 数据类型 如果变量属于该数据类型，返回true,否则返回false

十四、接口

接口，是Java语言中一种引用类型，是方法的集合，如果说类的内部封装了成员变量、构造方法和成员方法，那么接口的内部主要就是封装了方法（抽象方法、默认方法、静态方法、私有方法）

接口的使用，它不能创建对象，但是可以被实现，一个实现接口的类，需要实现接口中所有的抽象方法，这是的类为抽象类

定义

public interface Name {
    // 抽象方法
    public abstract void method1(); // 可以省略为 void method();
    // 默认方法
    public void method2(){
        //执行语句
    }
    // 静态方法
    public static void method3(){
        // 执行语句
    }
    private void method4() {

    }
    // 私有方法
}

基本实现

实现使用implements关键字

非抽象子类实现接口，必须重写接口中所有的抽象方法，继承了接口的默认方法，既可以直接调用，也可以重写。

class ClassName implements InterFaceName {
    // 重写接口中抽象方法
    // 重写接口中默认方法[可选]
}

抽象方法的使用

抽象方法必须全部实现。

静态方法的使用

接口内的静态方法只能通过接口来调用，不可以通过实现类的类名或者实现类的对象调用。

私有方法的使用

私有方法只有默认方法可以调用；私有静态方法只有默认方法和静态方法可以调用。

接口的多实现

一个类只能继承一个父类，而对于接口而言，一个类是可以实现多个接口的。

class ClassName extends FuClassName implements InterfaceName1,InterfaceName2...{
    // 重写接口中的抽象方法
    // 重写接口中的默认方法[可选]
}

抽象方法的实现

实现类必须重写所有的抽象方法，如果抽象方法有重名的，只需要重写一次。

默认方法的实现

接口中，有多个默认方法时，实现类都可继承使用，如果默认方法有重名的，必须重写一次。

静态方法的实现

接口中，存在同名的静态方法并不会冲突，原因只是通过各自接口名访问静态方法。

优先级问题

当一个类既继承了一个父类，又实现了若干个接口时，父类中的成员方法与接口中的默认方法重名，子类就近选择执行父类的成员方法。

接口的多继承

一个接口能继承另一个或者多个接口，这和类之间的继承比较相似。接口的继承使用 extends 关键字，子接口继
承父接口的方法。如果父接口中的默认方法有重名的，那么子接口需要重写一次。

其他成员特点

  • 接口中，无法定义成员变量，但是可以定义常量，其值不可以改变，默认使用public static final修饰
  • 接口中，没有构造方法，不能创建对象。
  • 接口中，没有静态代码块。

十五、final关键字

final 关键字代表最终的，不可改变的。

对于类、方法来说，abstract 和 final 不能同时使用。

修饰一个类

public final class 类名 {

}

被修饰的类不能被继承，其中所有的成员方法都无法进行覆盖重写。

修饰一个方法

修饰符 final 返回值类型 方法名(参数列表) {
    // 方法体
}

被修饰的方法不能重写

修饰一个局部变量

  • 基本类型 值不能更改
  • 引用类型 地址不可变，但是内部成员变量的值可变。

修饰一个成员变量

涉及到初始化问题，只能在定义时初始化，或在构造方法中初始化。

十七、内部类

将一个类A定义在另一个类B里面，类A叫做内部类，类B叫做外部类

成员内部类

定义在类内方法外的类，在描述事物时，若一个事物内部还包含其他事物，就可以使用内部类这样的结构。

访问特点

  • 内部类可以直接访问外部类的成员，包括私有成员
  • 外部类要访问内部类，必须要建立内部类的对象。 创建格式外部类名.内部类名 对象名 = new 外部类型().new 内部类型()
  • 内部类仍然是一个独立的类，在编译之后会内部类会编译成独立的class文件，但是前面冠以外部类的类名和$符号。

匿名内部类

它是内部类的简化写法，它的本质是一个带具体实现的父类或者父类接口的匿名子类对象。

匿名内部类必须继承一个父类或者实现一个父接口。

/*
new 父类名或接口名() {
    // 方法重写
    @override
    public void method() {
        // 执行语句
    }
}
*/

// 定义一个抽象类
public abstract class FlyAble {
    public abstract void fly();
}

public class InnerDemo {
    public static void main(String args) {
        // 等号右边是匿名内部类，定义并创建了抽象类的子类对象
        // 等号左边是多态赋值，抽象类引用指向子类对象。
        FlyAble f = new FlyAble() {
            public void fly() {
                System.out.println("︿(￣︶￣)︿")
            }
        }

        // 调用fly方法，执行重写后的方法
        f.fly();
    }
}

通常在方法的形式参数是接口或者抽象类时，也可以将匿名内部类作为参数传递。

public class InnerDemo3 {
    public static void main(String[] args) {
        /*
        创建匿名内部类,直接传递给showFly(FlyAble f)
        */
        showFly( new FlyAble(){
            public void fly() {
                System.out.println("我飞了~~~");
            }
        });
    } 
    public static void showFly(FlyAble f) {
        f.fly();
    }
}

String类

概述

Java程序中所有的字符串文字都可以被看作是实现此类的实例（程序当中所有的双引号字符串，都是String类的对象）

特点

  • 字符串不变：字符串的值再创建后不能被更改。
  • 因为String对象是不可变的，所以他们可以被共享。
  • 字符串的底层是用char数组来实现的。

使用

  • 不需要导入
  • 构造方法
    • public String() : 创意一个空白字符串
    • public String(char[] value): 根据当前参数的字符数组构造字符串
    • public String(bytep[] bytes)：根据字节数组内容来构造字符串
  • 注意：程序当前直接写的双引号字符串就在字符串常量池中，new的不在常量池中。

常用方法

判断功能的方法

  • public boolean equals(Object anObject): 将此字符串内容与指定对象进行比较
  • public boolean equalsIgnoreCase(String anotherString): 将此字符串与指定对象进行比较，忽略大小写

获取功能的方法

  • public int length(): 返回此字符串的长度
  • public String concat (String str): 将指定字符串连接到该字符串的末尾
  • public char charAt(int index): 返回指定索引处的char值
  • public int indexOf(String str): 返回指定子字符第一次出现在该字符串内的索引
  • public String substring(int beginIndex): 返回一个子字符串，从beginIndex开始截取字符串到字符串结尾。
  • public String substring(int beginIndex,int endIndex):返回一个子字符串，从beginIndex开始截取字符串到endIndex，包含beginIndex，但不包含endIndex。

转换功能的方法

  • public char[] toCharArray(): 将此字符串转换为新的字符数组
  • public byte[] getBytes(): 使用平台的默认字符集将该String编码转换为新的字节数组
  • public String replace(CharSequence target, CharSequence replacement): 将与target匹配的字符串使用replacement字符串替换。

分割功能的方法

  • public String[] split(String regex): 将此字符串安装给定的regex（规则）拆分为字符串数组。
